# 출시 범위 제품 결정 정본 (2026-07, QA4 확정)

> QA 4차 수렴에서 확정된 **제품 결정**을 기록한다. 아래 항목은 "미구현 결함"이 아니라
> **의도된 출시 계약**이며, QA 체크리스트·감사 문서는 이 문서를 정본으로 참조한다.
> (참조: `docs/FEATURE_AUDIT.md` §8 설계상 제외, `docs/APP_FEATURE_STATUS.md`)

## 1. 웹 전용 기능 (앱에서 제공하지 않음 — 결함 아님)

| 기능 | 결정 | 비고 |
|---|---|---|
| 프로필 아바타 교체 | **웹 전용** | 앱은 표시만. 업로드·교체 UI 미제공 |
| 멘토 인증 서류 업로드/관리 | **웹 전용** | 학생증·재학증명 등 민감 서류. 앱은 업로드 UI 미제공이며 **민감 서류 URL 을 앱 표면에 노출하지 않는다** |
| 결제·캐시 충전·구독 시작 | **웹 전용 (Commerce-Zero)** | 앱에는 구매 유도 CTA 없음. `CommerceNoticeCard` 비상호작용 안내 유지, `kInAppPaymentSteeringEnabled=false` 유지. Toss/Play Billing/in_app_purchase/결제 WebView 금지 |

## 2. 로그인 단순 스코프 = 의도된 출시 범위

- 앱 로그인은 **이메일/비밀번호 + 둘러보기(게스트)** 만 제공한다.
- 회원가입은 **웹 가입 페이지**로 연결한다(앱 내 가입 폼 없음).
- 소셜 로그인·온보딩 플로우는 이번 출시 범위에 **포함하지 않는다**.
- QA 문서에서 이를 "미구현 결함"으로 기록하지 않는다 — 오너 확정 출시 계약이다.

## 3. 커뮤니티 게시판 작성기 — 웹 작성 표면이 정본

- 게시판 글 작성은 웹 작성기(정본)를 사용한다. **제거·WebView 전면 재설계 금지.**
- 숏폼 업로드는 별도 확정된 앱 WebView 브릿지(`/app/bridge/complete` 계약)를 따른다.

## 4. 맞춤의뢰(CR) 게이트 OFF — 서버 계약 ≠ 앱 출시 표면

- 앱은 CR 알림 2종(`new_order_message`, `new_application`)을 **exact type 비교**로
  표시하지 않는다(부분 문자열 매칭 아님). DB 쿼리 단계 제외로 keyset
  (created_at, id) 커서 무결성을 유지한다.
- **서버 producer·DB 정본 17종 enum·발신 계약은 불변.** "서버가 보낼 수 있는 유형
  집합(계약)"과 "앱이 이번 출시에서 노출하는 표면"은 서로 다른 개념이다.
- 알림 설정 라벨은 `개별질문 알림`(서버 key `order` 는 호환 유지).
- CR 필터 칩·카테고리·딥링크는 추가하지 않는다.
- **한계(보고 사항)**: 서버 푸시 발신은 변경하지 않으므로, CR 이벤트의 FCM 푸시가
  시스템 트레이에 표시되는 것 자체는 앱 단에서 억제할 수 없다(푸시 인프라 미가동
  상태에서는 실영향 없음). 인앱 목록·이동은 이미 제외/무동작이다.

## 5. 리뷰 메뉴 역할 게이트

- 학생·게스트·관리자에게 범용 `리뷰 작성` 행을 **노출하지 않는다**.
- 멘토에게만 `받은 리뷰` 행을 노출하며, 목적지는 웹 `/mentor/reviews` 이다.
- 앱 내 리뷰 작성 컴포저는 만들지 않는다(리뷰 작성 조건: 동일 멘토 2회 연속 결제
  성공 — 웹 정본 플로우).

## 6. 버튼 색 위계

- **액션 CTA = 고정 파랑 `#2563EB`** (역할 무관): `PrimaryButton`, 액션
  `SecondaryButton`, 받은 질문 보기·저장·로그아웃·FAB 등.
- **멘토 정체성 초록(`AppAccent`, #059669)은 배지·탭·장식에 유지** — 액션 색으로
  쓰지 않는다. 전면 초록 치환 금지.
- 위험(danger)·중립(neutral) 계열은 불변.
