# MANUAL_QA_CHECKLIST — 실기기 수동 검증 체크리스트

> 헤드리스 컨테이너(위젯 테스트)로 검증 불가한 항목 모음. QA_REPORT_2026-07.md 의 자동 검증을 보완한다.
> 기기 권장 매트릭스: Android 폰(중저가 1대 포함) · Android 태블릿+S펜 · iPhone · iPad+Apple Pencil.
> 각 항목은 [ ] 체크박스 + 기기/OS 기록란을 채워서 사용.

## 1. 스타일러스·필기 (S15 스캔 첨삭 — 태블릿 필수)

- [ ] **팜 리젝션(펜 전용 모드)**: 주석 화면 기본 모드에서 손바닥을 캔버스에 대고 펜으로 필기 — 손바닥 터치가 스트로크를 만들지 않는가. (iPad+펜슬 / 갤럭시탭+S펜 각각)
- [ ] **손가락 토글**: 툴바에서 손가락 그리기 허용으로 전환 시 손가락 스트로크가 그려지고, 다시 펜 전용으로 돌리면 무시되는가.
- [ ] **줌·팬과 필기 공존**: 두 손가락 핀치 줌/팬 중 의도치 않은 스트로크가 생기지 않는가. 줌 상태에서 그린 선이 확대 배율과 무관하게 이미지의 같은 위치에 붙는가(좌표 정합).
- [ ] **기기 간 좌표 정합**: 폰에서 첨삭해 전송한 이미지를 태블릿에서 열었을 때 선 위치가 원본 대비 어긋나지 않는가(반대 방향도).
- [ ] **평탄화 정확도**: 전송된 PNG에서 선 굵기·색·위치가 편집 화면과 동일한가. 고해상 사진(4000px+)에서도 깨지지 않는가.
- [ ] **필압**(P1 예정 기능의 사전 확인): 현재는 압력→선폭 매핑 off — 필압을 바꿔도 선폭이 일정한가(의도된 동작).
- [ ] **undo/redo·전체 지우기**: 50+ 스트로크에서 지연 없이 동작하는가.

## 2. 푸시 알림 (FCM 인프라 구축 후)

- [ ] 포그라운드/백그라운드/종료 상태 각각에서 수신되는가.
- [ ] 알림 탭 → 앱 열림 → 해당 탭으로 이동하는가: 질문방 알림→질문방 탭, 구독 알림→마이페이지(push), 개별질문 알림→개별질문 탭.
- [ ] **콜드스타트 딥링크**: 앱이 완전히 종료된 상태에서 알림 탭 → 스플래시 → 로그인 상태 복원 → 목적지 도달 순서가 깨지지 않는가(로그인 화면으로 새는지 확인).
- [ ] 알림 권한 거부 후 "알림 다시 켜기"(마이페이지 설정) → OS 설정 유도가 동작하는가.
- [ ] 기기 토큰: 로그아웃/재로그인·앱 재설치 후 다른 계정의 알림이 오지 않는가.

## 3. 카메라·갤러리 권한 (스캔 소스 — S16 구현 후 확장)

- [ ] 갤러리 선택: 권한 최초 요청 → 허용/거부/"다시 묻지 않음" 각각에서 앱이 죽지 않고 안내가 뜨는가.
- [ ] iOS '선택한 사진만' 제한 모드에서 첨부가 동작하는가.
- [ ] **HEIC**(아이폰 기본 포맷) 업로드 → 상대 기기(Android)에서 정상 표시되는가.
- [ ] 5MB 초과·고해상 사진 업로드 시간·리사이즈 품질이 수용 가능한가.
- [ ] (S16 카메라 추가 후) 촬영 권한 거부 시 폴백, 저조도·기울어진 문서 촬영 품질.

## 4. 키보드·IME (한글 입력)

- [ ] 채팅 입력바: 한글 조합 중(자모 조합 상태) 전송 버튼을 눌러도 마지막 글자가 유실되지 않는가.
- [ ] 질문 작성·연결노트: 긴 글 입력 중 키보드가 화면을 가리지 않는가(스크롤 따라감), 키보드 열림/닫힘에 RenderFlex overflow가 없는가(특히 320~360dp 소형 기기 — 위젯 테스트는 320×568 통과, 실기기 IME 패널은 별도).
- [ ] 이모지·특수문자(수식 √∑≤)가 입력·표시·전송 전 과정에서 깨지지 않는가.
- [ ] 외장 키보드(태블릿): Enter=전송/줄바꿈 동작이 의도대로인가.

## 5. 로그인·계정 플로우 (실 Supabase)

- [ ] 이메일+비밀번호 로그인 성공/오답/미가입 각각의 안내 문구(내부 에러 원문이 노출되지 않는가 — QA-02 확인 겸용).
- [ ] 세션 복원: 앱 강제 종료 → 재실행 시 로그인 유지, 만료 토큰 자동 갱신.
- [ ] 게스트 둘러보기 → 커뮤니티·멘토찾기만 접근, 보호 탭 탭 시 로그인 안내 → 로그인 후 원래 자리로 복귀하는가.
- [ ] **차단 계정**(banned/suspended)·**admin 계정** 로그인 → 차단 화면, 뒤로가기로 우회 불가.
- [ ] 로그아웃 → 로그인 화면, 뒤로가기로 이전 화면 복귀 불가.
- [ ] 회원가입 링크 → 웹 가입 페이지가 열리는가(웹 도메인 확정 후).

## 6. 결제·웹 브릿지 동선 (Commerce-Zero — 웹 도메인 확정 후)

- [ ] 구독 관리/충전·결제/정산 관리/약관/개인정보/회원탈퇴 각 버튼 → **외부 브라우저**로 열리는가(인앱 웹뷰 아님).
- [ ] 열린 웹 페이지가 올바른 경로(`/subscriptions` 등)+`src=app` 쿼리로 도달하는가.
- [ ] 앱 내 어디에도 구매 유도 CTA·상품 가격이 노출되지 않는가(멘토 카드·상세·IQ 카드 — 스토어 심사 관점 재확인).
- [ ] 웹에서 결제 후 앱 복귀 → 구독 상태가 새로고침되는가(수동 새로고침 필요 여부 기록).

## 7. 실시간·네트워크 상태

- [ ] 채팅: 두 기기 로그인(학생·멘토) → 메시지가 상대에게 실시간 도착하는가(realtime publication 미설정 시 수동 새로고침 폴백 동작).
- [ ] 비행기 모드에서 전송/저장 시도 → 앱이 죽지 않고 재시도 안내, 복귀 후 정상 동작.
- [ ] 느린 3G 프로파일에서 목록 로딩 스피너·타임아웃 UX.

## 8. 접근성·표시

- [ ] 시스템 글자 크기 최대(접근성 배율)에서 하단 탭·카드·버튼 오버플로 없는가.
- [ ] 다크 모드(지원 안 하면 라이트 고정이 유지되는가 — 중간 상태로 깨지지 않는지).
- [ ] TalkBack/VoiceOver: 하단 탭·프로필 버튼(시맨틱 라벨 '마이페이지')·필기 툴바가 읽히는가.

## 9. 기능 스위치 스모크 (릴리즈 빌드 전)

- [ ] `kIndividualQuestionCreateEnabled=false` 빌드: 개별질문 탭에 목록·상세는 유지되고 '새 개별질문'·멘토 상세 '개별질문 하기'가 사라지는가. (컴파일 상수라 자동 테스트 불가 — QA_REPORT §검증 통과 항목 참고)
- [ ] `kIndividualQuestionEnabled=false` 빌드: 탭은 유지되되 '준비 중' 안내만 뜨는가.
