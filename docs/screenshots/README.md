# 앱 현재 화면 스크린샷 (캡처 전용)

> 정본 앱(`byite1226-a11y/ssambership-app`)을 **현재 상태 그대로** 실행해 캡처했습니다.
> **색은 지금의 스카이 placeholder 그대로**입니다(확정색 파랑 `#2563EB` 미적용 — 재디자인은 목업에서).
> 코드·토큰·데이터 변경 없음. 캡처를 위한 임시 빌드 스캐폴딩(`web/`)·로컬 빈 `.env`만 사용했고 **커밋하지 않았습니다.**

## 캡처 환경
- Flutter 웹 빌드(profile)를 로컬에서 렌더 → 모바일 세로(**390×844, DPR 2**)로 헤드리스 크롬 캡처.
- **백엔드(Supabase) 미연결** 상태(키 없음). 앱 설계상 이때는 '빈 앱'이라, 데이터가 필요한 화면은 **빈/에러 상태**로 나옵니다(가짜 데이터 없음).
- 앱에 데모/시드/mock 데이터 경로는 **존재하지 않음**(코드 확인). 그래서 콘텐츠가 채워진 상태는 백엔드 연결 없이는 캡처 불가.
- 한글 폰트(Noto Sans KR)는 런타임에 CDN에서 받는 구조라, 캡처 시 로컬 폰트로 대체해 글자가 보이도록 했습니다(앱 자체 변경 아님).

## 못 찍은 화면과 이유

아래는 **로그인 세션 + 실데이터(Supabase)** 가 있어야 도달하는 화면들입니다. 이번은 **캡처 전용(백엔드 미연결·코드 변경 없음)** 이라, 앱에 데모/시드 경로가 없어 캡처 불가했습니다.

| 요청 화면 | 이유 |
|---|---|
| **온보딩** | 화면 파일은 존재(`lib/features/onboarding/onboarding_screen.dart`)하나 **라우터/플로우에 미연결** → 정상 경로로 도달 불가 |
| **학생 홈(질문방 탭) / 질문방 목록·스레드(채팅)·연결노트** | 로그인 필요(게스트 접근 불가). 스레드·연결노트는 실데이터 필요 |
| **멘토 상세** | 멘토 목록이 비어(백엔드 미연결) 상세로 진입할 항목이 없음 |
| **커뮤니티 상세(숏폼/게시글)·댓글** | 목록이 비어 상세 진입 불가 |
| **마이페이지**(구독현황·캐시조회·설정) | 로그인 필요 |
| **알림 센터** | 로그인 필요 |
| **멘토 역할 화면(답변 작성 등)** | 멘토 로그인 + 실데이터 필요 |

> 위 화면들의 **레이아웃/디자인 감**은 `06_dev_widget_gallery_*` 의 컴포넌트(빈 상태·카드·상태 pill 등)로 일부 확인 가능합니다.
> 실제 콘텐츠가 채워진 화면 캡처가 필요하면, **로컬 Supabase 시드 연결(라이브 연결 단계)** 후 다시 캡처하면 됩니다.
