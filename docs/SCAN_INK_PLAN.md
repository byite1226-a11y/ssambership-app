# 스캔 필기(Scan-Ink) 기능 기획안

> 문제를 사진·파일로 스캔하고, 그 위에 손가락·스타일러스(Apple Pencil / S펜)로 그린 필기를 렌더링하는 기능.
> 배치 위치: **질문방 + 개별질문**. 연결노트의 자유 캔버스 필기는 오구현으로 판단하여 **제거 완료**(§9 참고).

작성일: 2026-07-06 · 상태: 승인 대기 · 선행 커밋: 하단 탭 개편 + 연결노트 필기 제거(본 문서와 같은 브랜치)

---

## 1. 배경과 문제 정의

기존 필기 기능(S14)은 연결노트 안의 **빈 캔버스 자유 필기**로 구현되어 있었다. 하지만 이 서비스에서 필기의 실제 쓰임새는 "메모를 손으로 쓰는 것"이 아니라, **문제(교재·시험지·프린트)를 스캔해서 올리고 그 위에 풀이·표시·첨삭을 얹는 것**이다. 즉 필기는 노트 기능이 아니라 **질문·답변 커뮤니케이션의 표현 수단**이며, 따라서 질문이 오가는 두 지점 — 질문방(구독 기반)과 개별질문(건당 캐시) — 에 있어야 한다.

다행히 코드베이스에는 이미 이 방향의 절반이 구현되어 있다. S15 스캔 주석(`lib/features/scan_annotation/`)이 "질문방 채팅의 이미지 첨부 위 첨삭"을 정확히 이 방식(이미지 기준 0..1 정규화 좌표 저장 → 원본 해상도 평탄화 PNG 전송)으로 처리한다. 본 기획은 **S15를 재구현 없이 일반화**하여 (1) 스캔 소스를 카메라·파일(PDF)로 확장하고 (2) 개별질문에 첨부·첨삭 파이프라인을 신설하는 것이 골자다.

## 2. 목표 / 비목표

**목표**

- 학생이 문제를 **카메라 촬영·갤러리 선택·파일(PDF) 업로드**로 스캔해 질문에 첨부할 수 있다.
- 첨부한 스캔 위에 **펜/지우개/색/굵기** 도구로 필기를 얹을 수 있다. 태블릿·스마트폰에서 손가락과 스타일러스 모두 지원하며, 스타일러스 사용 시 팜 리젝션(펜 전용 모드)이 동작한다.
- 필기 스트로크는 **이미지 기준 정규화 좌표(0..1)** 로 저장되어 기기·해상도·줌과 무관하게 위치가 보존되고, **재편집(이어 그리기)** 이 가능하다.
- 전송 시에는 배경 원본 해상도로 **평탄화(flatten)된 PNG** 가 상대방에게 전달된다 — 수신 측은 특별한 뷰어 없이 일반 이미지로 본다.
- 멘토는 학생이 올린 스캔 위에 **첨삭(빨간펜)** 을 얹어 답변으로 보낼 수 있다 (질문방·개별질문 동일).

**비목표 (이번 범위 제외)**

- 자유 캔버스 노트(빈 종이에 쓰는 필기) — 연결노트에서 제거했고 되살리지 않는다.
- 벡터 상태 그대로의 실시간 공동 편집(동시 첨삭), 필기 인식(OCR/수식 인식), 형광펜·도형·텍스트 박스(P1 후보), 녹화형 풀이 재생.

## 3. 배치 위치와 진입점

| 위치 | 진입점 | 동작 |
|---|---|---|
| 질문방 — 새 질문/채팅 | 첨부 버튼 → **촬영·갤러리·파일** 선택 → 미리보기의 "필기하기" | 전송 전 스캔 위에 필기 후 평탄화 PNG 첨부 (기존 S15 흐름 확장) |
| 질문방 — 받은 이미지 | 말풍선 이미지 → 전체화면 뷰어 → "필기하기" | 상대가 보낸 스캔 위에 첨삭 → 새 첨부로 전송 (기존 S15 유지, 라벨만 '주석 달기'→'필기하기') |
| 개별질문 — 작성(학생) | 작성 화면에 첨부 영역 신설 → 스캔 → "필기하기" | 문제 스캔 + 풀이 시도·모르는 부분 표시를 얹어 질문 등록 |
| 개별질문 — 답변(멘토) | 질문 상세의 학생 첨부 → "첨삭하기" | 학생 스캔 위에 첨삭 → 답변 첨부로 전송 |

개별질문이 하단 탭으로 승격되었으므로(§9), 필기의 두 진입점(질문방·개별질문)이 모두 1급 동선에서 한 번의 탭으로 닿는다.

## 4. 사용자 시나리오

**학생 A (스마트폰, 손가락)** — 수학 문제집을 카메라로 촬영한다 → 자동 보정(기울기·여백 크롭은 P1, 1차는 원본 그대로) → "필기하기"로 들어가 모르는 부분에 동그라미와 물음표를 그린다 → 전송. 멘토에게는 필기가 합성된 한 장의 이미지가 도착한다.

**학생 B (태블릿 + 펜슬)** — 학원 프린트 PDF를 파일로 업로드한다 → 질문할 페이지를 고른다(다중 페이지는 페이지당 1장으로 분리) → 펜 전용 모드가 자동 활성화되어 손바닥을 대고 필기해도 오입력이 없다 → 풀이 과정을 페이지 위에 쓰고 막힌 줄에 별표를 친 뒤 개별질문으로 등록한다.

**멘토 C** — 개별질문 탭에서 수락한 질문을 연다 → 학생의 스캔+필기 이미지를 열고 "첨삭하기" → 빨간 펜으로 틀린 전개를 긋고 옆에 올바른 식을 쓴다 → 답변 전송. 학생은 답변 도착 푸시를 받고 첨삭본을 확인한다.

## 5. 핵심 플로우

```
[스캔 소스]                [편집]                          [저장/전송]
카메라 촬영 ─┐
갤러리 선택 ─┼→ 이미지 확정 → 필기 캔버스(이미지와 동일   → ① 원본 스트로크 ink.json 업로드(재편집용)
파일(PDF)  ─┘   (페이지→PNG)   사각형에 오버레이, 줌·팬     ② 원본 해상도 평탄화 PNG 를 첨부로 전송
                              동기화, 펜/손 입력 모드)
```

- **재편집**: 전송 전이라면 ink.json 을 내려받아 스트로크를 복원해 이어 그린다. 전송 후 첨삭은 "받은 이미지 위에 새 레이어"로 취급한다(원본 훼손 없음, 새 첨부 생성).
- **좌표 정합(불변 규약)**: 스트로크는 화면 픽셀이 아니라 이미지 기준 0..1 정규화 좌표로 저장한다. 저장 직전 `InkCoordinateMapper` 로 normalize, 복원 시 denormalize. 평탄화는 `AnnotationFlattener` 가 배경 원본 픽셀 크기 캔버스에 다시 그린다. **이 규약은 S15 에서 검증 완료 — 변경 금지.**

## 6. 화면 설계

**6-1. 스캔 소스 선택 시트** — 첨부 버튼 탭 시 바텀시트 3택: `촬영` / `갤러리` / `파일`. 파일은 이미지(jpg·png·webp·heic)와 PDF 를 허용한다. PDF 선택 시 페이지 썸네일 그리드에서 질문할 페이지를 고르면 해당 페이지를 PNG 로 래스터화해 이미지 파이프라인에 태운다(페이지당 1첨부, 최대 5페이지/회).

**6-2. 필기 캔버스 화면** — 기존 `ScanAnnotationScreen` 을 그대로 쓴다. 상단 앱바(취소·완료), 중앙 InteractiveViewer(줌·팬, 이미지와 캔버스가 같은 레이어라 추가 보정 불필요), 하단 `InkToolbar`(펜/지우개 · 색 3종(검정·빨강·파랑) · 굵기 3단 · undo/redo · 전체 지우기 · 손가락 그리기 토글). 멘토 첨삭 진입 시 기본 색을 빨강으로 프리셋한다.

**6-3. 입력 모드(팜 리젝션)** — `InkInputMode` 정책을 그대로 사용한다. 기본은 `penOnly`(스타일러스 포인터만 스트로크로 인식 → 손바닥 무시), 툴바 토글로 `fingerAllowed` 전환. 스타일러스 압력값은 scribble 포인트의 pressure 로 이미 저장되므로, 압력→선폭 매핑은 어댑터 옵션 추가만으로 켤 수 있다(P1, 기본 off).

**6-4. 상태·예외** — 업로드 실패 시 스트로크는 로컬에 유지한 채 재시도 안내(작업 유실 금지). 5MB 초과 스캔은 업로드 전 리사이즈(image_picker `imageQuality: 85` 유지 + 장변 4096px 캡). PDF 래스터화 실패·암호화 PDF 는 "이미지로 촬영해서 올려 주세요" 폴백 안내.

**7-1. 재사용 자산 (재구현 금지)**

| `ScanAnnotationRepository` + `SupabaseAttachmentUploader` | 〃 + `question_room/data/attachments/` | ink.json 저장 + 평탄화 PNG 첨부 전송 파이프라인 |

**7-2. 신규 모듈**

- `lib/core/scan/scan_source_picker.dart` — 촬영·갤러리·파일 3소스 통합 포트. 촬영은 기존 `image_picker` 의 `ImageSource.camera` 로 커버(신규 의존성 없음). 파일은 `file_picker` 도입.
- `lib/core/scan/pdf_rasterizer.dart` — PDF 페이지 → PNG. 후보: `pdfx`(플러그인, iOS/Android 네이티브 렌더) 우선, 불가 시 P0 는 이미지 스캔만 출시하고 PDF 는 P1 로 미룬다(§11 리스크).
- `lib/features/individual_question/data/iq_attachments_repository.dart` — 개별질문 첨부 업로드/조회. 업로더 포트는 질문방 것을 재사용하되 버킷·경로만 다르다.
- `ScanAnnotationScreen` 일반화 — 현재 시그니처가 `roomId`/`threadId` 에 결합되어 있으므로, 전송 대상을 포트로 추상화한 `AnnotationTarget`(질문방 스레드 첨부 / 개별질문 첨부)을 주입받도록 **옵션 추가**(기존 호출부 무영향).

## 9. 이번 커밋에 반영된 선행 변경 (완료)

1. **하단 탭 개편** — `질문방 · 커뮤니티 · 멘토 찾기 · 알림 · 개별질문` 5탭. 개별질문이 다른 기능과 동일 위상으로 승격 (`home_shell.dart`, `app_tabs.dart`, `app_constants.dart`).
2. **마이페이지 이동** — 하단 탭에서 제거하고 AppBar **우측 상단 원형 도형 속 프로필 아이콘** 버튼으로 진입(push). 알림 딥링크의 `AppTab.myPage` 는 가상 목적지(=100)로 유지되어 기존 호출부 호환.
3. **개별질문 탭 화면** — `IndividualQuestionTabScreen` 신설, role 에 따라 기존 학생/멘토 목록 화면을 `embedded` 모드로 재사용. 마이페이지의 중복 진입 섹션은 제거.
4. **연결노트 필기 제거** — `connection_notes_screen` 에서 필기 버튼·썸네일·저장 흐름 삭제, `ink_note/` 모듈(화면·캔버스·레포·결과) 삭제. 공용 `InkToolbar` 만 `lib/core/ink/widgets/` 로 이동해 스캔 첨삭(S15)이 계속 사용. 관련 테스트 3건 삭제, 툴바 테스트는 `test/core_ink/` 로 이동.

## 10. 테스트 계획

좌표 정합(정규화 왕복·평탄화 픽셀 위치)은 기존 S15 테스트가 커버하므로 회귀 유지만 확인한다. 신규로는 (a) 소스 선택 시트 → 각 소스 포트 호출 검증(fake picker 주입), (b) IQ 첨부 업로드 경로·RLS 경계(질문 당사자 외 접근 거부)의 레포 테스트, (c) `AnnotationTarget` 포트가 질문방/IQ 각각의 업로더를 올바르게 태우는지, (d) 펜 전용 모드에서 터치 포인터가 스트로크를 만들지 않는지(기존 InkInputMode 테스트 확장), (e) PDF 래스터화 실패 폴백 안내를 검증한다.

## 11. 리스크와 오픈 이슈

- **PDF 렌더 의존성** — `pdfx` 는 네이티브 바인딩이라 플랫폼별 검증 비용이 있다. S19 를 독립 단계로 분리한 이유이며, 일정 압박 시 P0 는 이미지 스캔만으로 출시 가능하다.
- **대용량 스캔** — 고해상 촬영 + 평탄화 PNG 는 수 MB 가 될 수 있다. 장변 4096px 캡과 품질 85 압축을 규약으로 두고, 초과 시 업로드 전 리사이즈한다.
- **iOS HEIC** — 기존 mime 추론에 heic 이 이미 포함되어 있으나, 평탄화 출력은 항상 PNG 이므로 수신 호환 문제는 없다.
- **첨삭 원본 보존 정책** — 멘토 첨삭이 학생 원본을 덮지 않고 항상 새 첨부를 만드는 규약을 유지할지, 같은 첨부의 레이어로 쌓을지(재편집 UX 향상 vs 복잡도)는 S18 착수 전 결정 필요. 본 기획의 기본안은 **항상 새 첨부**다.
